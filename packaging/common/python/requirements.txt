# TODO: publish the pure-Python pyproto exporter chain to PyPI and replace the
# three vendored requirements below with a single pinned release, e.g.:
#   opentelemetry-exporter-otlp-pyproto-http==<version>
#
# Until published, the exporter and its two pyproto-only dependencies are
# vendored under vendor/ — see vendor/README.md for provenance and the sync
# procedure. They are pure-Python (no google-protobuf C extension) and are
# installed from source by the build's source pass with --no-deps; their only
# dependencies beyond each other (opentelemetry-api, opentelemetry-sdk) are
# covered by the pins above. The exporter transports via urllib and needs no
# HTTP client dependency.
#
# The exporter is a drop-in replacement for opentelemetry-exporter-otlp-proto-http:
# it owns the public opentelemetry.exporter.otlp.proto.http module path (a shim
# over the private pure-Python implementation) and registers the otlp_proto_http
# entry points. This also covers declarative configuration (the SDK's file
# configurator resolves the otlp_http exporter type through exactly that module
# path), so the real proto-http package and its google-protobuf C extension must
# NOT be bundled — the two distributions ship conflicting files.
# The vendored pyproto-grpc exporter is not shipped: it requires grpcio, a C
# extension, which would break the pure-Python property of this bundle.
./vendor/opentelemetry-exporter-otlp-pyproto-http
./vendor/opentelemetry-exporter-otlp-pyproto-common
./vendor/opentelemetry-pyproto
